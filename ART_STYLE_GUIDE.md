# MultiRough 原创美术规范

版本：1.0  
状态：职业角色第一版制作基准

## 1. 目标

本项目采用原创的 2D 像素美术。所有正式角色、敌人、场景、特效和 UI 均应按照本规范重新设计，不直接描摹、改色、拼接或修改 Tiny Swords 等第三方素材。

本规范优先保证：

- 多人战斗中角色轮廓清晰。
- 五个职业能够通过轮廓和武器识别，而不只依赖颜色。
- 所有动画可以直接接入 Godot 当前的横向精灵表流程。
- 缩放后保持像素边缘清晰，不产生模糊或半像素抖动。

## 2. 像素密度与画布

### 2.1 动画单元

- 每帧固定画布：`192 × 192 px`。
- 动画帧沿水平方向排列。
- PNG 使用透明背景和 RGBA 色彩。
- 默认角色朝向右侧；左侧由程序水平翻转。
- 同一职业的所有动画必须使用相同画布、脚底基准线和角色中心点。

### 2.2 角色实际尺寸

角色不应填满 192 × 192 画布：

- 常规身体高度：`78–92 px`。
- 常规身体宽度：`42–58 px`。
- 重型职业最大身体宽度：`64 px`。
- 头部高度约占身体总高度的 `28–34%`。
- 武器、披风和施法动作允许超出身体范围，但必须留在画布内。
- 脚底中心固定在 `(96, 138)`，误差不得超过 1 px。
- 身体视觉中心建议位于 `(96, 98)` 附近。

这些尺寸在当前约 `0.55` 的游戏缩放下，对应约 43–51 个屏幕像素的角色高度。

### 2.3 像素规则

- 以 1 px 为最小绘制单位，不使用抗锯齿笔刷。
- 不使用半透明像素模拟轮廓抗锯齿。
- 不使用亚像素移动制作动画。
- 斜线优先使用规律的 1:1、1:2 或 2:3 像素阶梯。
- 纹理导入必须使用 Nearest/最近邻过滤。
- 游戏内缩放应保持统一；单个动画不得临时改变角色缩放。

## 3. 角色比例和轮廓

### 3.1 基础比例

- 角色采用约 `2.5–3 个头身` 的夸张比例。
- 头部略大，以保证缩小后仍能识别职业。
- 手、武器和肩部允许适度放大，优先保证动作可读性。
- 双脚之间至少保留 3 px 间隔，避免移动动画缩成一团。

### 3.2 五职业轮廓要求

| 职业 | 轮廓重点 | 主要武器 | 禁止仅靠颜色表达的特征 |
|---|---|---|---|
| 战士 | 宽肩、稳定重心、护甲块面 | 单手剑与盾 | 盾牌必须进入待机轮廓 |
| 弓箭手 | 轻装、较窄身体、背部箭袋 | 长弓 | 弓或箭袋必须持续可见 |
| 长枪 | 纵向延伸、站姿挺直 | 长枪 | 枪身长度至少为身体高度的 1.15 倍 |
| 法师 | 长袍下摆、帽兜或高领、宽袖 | 法杖或法器 | 法杖顶部必须形成独特外轮廓 |
| 刺客 | 低重心、前倾、披巾或短斗篷 | 双短刃 | 两把短刃或非对称刀鞘必须可见 |

角色剪影填充为纯黑时，仍应能够分辨五个职业。

## 4. 描边规范

- 外轮廓使用 2 px 深色描边。
- 小型内部结构使用 1 px 描边。
- 不使用纯黑 `#000000` 作为主轮廓。
- 通用深色轮廓：`#182033`。
- 暖色物体可使用偏暖轮廓：`#2A1D25`。
- 冷色或金属可使用偏蓝轮廓：`#18263A`。
- 受光侧允许局部将外轮廓减为 1 px 或改成材料深色。
- 阴影侧必须保持连续轮廓，避免角色与地面混合。
- 内部线条应少于外轮廓，禁止把每个色块都描边。

## 5. 光照与阴影

### 5.1 固定光源

- 全局主光源来自画面左上方。
- 光照方向固定为左上到右下。
- 角色左上侧使用亮色，右下侧使用暗色。
- 所有角色、敌人、场景和 UI 插画遵守相同光源。

### 5.2 明暗层级

每种主要材质原则上使用：

1. 最深阴影
2. 基础色
3. 受光色
4. 少量高光（仅金属、魔法和光滑材料）

禁止依赖平滑渐变塑造身体。渐变只允许用于独立特效，并应控制在少量透明层中。

### 5.3 地面投影

- 角色地面影子使用独立椭圆或阴影节点，不画进角色动画。
- 推荐尺寸：`48 × 18 px`。
- 影子中心对齐脚底中心。
- 颜色：`#111827`，建议游戏内透明度 `28–36%`。
- 跳跃、冲刺或浮空技能可以改变影子大小，但不得改变脚底基准点。

## 6. 基础色板

正式资源优先从以下基础色板扩展。单个角色建议控制在 16–24 个不透明颜色以内。

### 6.1 通用颜色

| 用途 | 色值 |
|---|---|
| 最深轮廓 | `#182033` |
| 冷暗部 | `#26344D` |
| 中性暗部 | `#3A465D` |
| 浅色高光 | `#F2E9D8` |
| 暖白 | `#FFF3D6` |
| 深肤色 | `#8F4F45` |
| 中肤色 | `#C97862` |
| 浅肤色 | `#F0B38F` |
| 深金属 | `#34435C` |
| 中金属 | `#71839E` |
| 亮金属 | `#C5D2DE` |
| 深皮革 | `#493044` |
| 中皮革 | `#7A4B47` |
| 浅皮革 | `#B87855` |

### 6.2 职业识别色

| 职业 | 主色 | 暗色 | 高光 |
|---|---|---|---|
| 战士 | `#3486C7` | `#21466F` | `#78C7E8` |
| 弓箭手 | `#4F9B62` | `#28543B` | `#9BD26E` |
| 长枪 | `#C36B32` | `#713C2D` | `#F0AE4B` |
| 法师 | `#7656C9` | `#3E316F` | `#B9A0FF` |
| 刺客 | `#A53C54` | `#4D263E` | `#E16A78` |

识别色只占角色面积的约 `25–40%`。其余部分使用皮革、金属、布料和肤色，避免角色成为单一纯色块。

### 6.3 魔法与战斗特效色

- 通用物理命中：`#FFD36A`。
- 治疗与生命：`#65D77B`。
- 法师奥术：`#9A7CFF` 与 `#6ED8FF`。
- 危险和敌方伤害：`#F05B55`。
- 刺客标记或暗影：`#D44C78` 与 `#573B82`。
- 特效中心可以接近白色，外缘必须逐级降低亮度和透明度。

## 7. 俯视角度与朝向

- 采用约 `55–60°` 的三分之四俯视角。
- 角色能够看到头顶、肩部和少量武器上表面。
- 不绘制完整正侧面，也不采用完全垂直的 90° 顶视图。
- 脸部朝向以右前方为默认，移动到左侧时水平翻转。
- 不单独制作上下左右四方向动画；第一版继续使用单方向动画加水平翻转。
- 脚底位置保持稳定，身体可以上下浮动 1–2 px 表现呼吸和步伐。
- 武器攻击轨迹必须与游戏中的命中方向一致，不得出现明显反向动作。

## 8. 动画交付规范

### 8.1 第一批必需动画

| 动画 | 建议帧数 | 循环 | 用途 |
|---|---:|---|---|
| Idle | 6–8 | 是 | 待机和选角展示 |
| Run | 6–8 | 是 | 地面移动 |
| Attack 1 | 4–6 | 否 | 普攻 |
| Attack 2 | 4–6 | 否 | Q/E 技能通用动作 |
| Cast | 5–8 | 否 | F 或持续施法 |

### 8.2 第二批补充动画

- Hit：2–3 帧。
- Death：6–8 帧，结束帧保持倒地。
- Dash：3–5 帧；若不制作，可先复用 Run。
- Defend：2–4 帧，战士和长枪优先。

### 8.3 文件命名

```text
assets/original/characters/mage/
  mage_idle.png
  mage_run.png
  mage_attack_1.png
  mage_attack_2.png
  mage_cast.png
  mage_hit.png
  mage_death.png
  mage_portrait.png
```

文件名统一使用小写英文和下划线。每张精灵表只包含一个动画。

## 9. 原创性要求

- 不在第三方精灵上改色、换头、拼接武器或描摹轮廓。
- 不复刻第三方角色的具体盔甲分块、面部、武器比例或动画关键帧。
- 可以采用通用的像素美术技术，但必须重新设计人物比例、服装、武器和动作关键姿势。
- 保存草图、分层源文件、迭代记录和导出日期，建立原创制作记录。
- 使用外部素材、笔刷、字体或生成工具时，单独记录来源和许可范围。
- 正式发布前检查项目中是否仍引用 `assets/tiny_swords_free_pack/`。

## 10. 首个角色验收标准

法师作为第一名原创样板角色，交付时必须满足：

- 五个第一批动画齐全。
- 每帧均为 192 × 192，脚底中心保持在 `(96, 138)`。
- 黑色剪影能够与战士、弓箭手和长枪区分。
- 外轮廓、光源和色板符合本规范。
- 缩放到游戏尺寸后，法杖、头部和手部动作仍可辨认。
- 精灵表不包含第三方像素或经过修改的第三方素材。
- 接入 Godot 后不需要为不同动画单独调整位置或缩放。

法师样板验收后，战士、弓箭手、长枪和刺客均使用相同的画布、脚底基准、光照和交付格式。
