# MultiRough 开发历史归档

> 本文件保存已经完成的阶段、被替换的方案和重要方向变化。
> 当前有效状态与下一步计划以仓库根目录的 `process.md` 为准。

## 2026-07-08：单人与本地双人原型

- 建立 Godot 4.7 俯视角清波 Roguelite 原型。
- 完成单人移动、普攻、防御、闪避、技能、波次、升级、Boss、胜利和失败闭环。
- 接入本地双人同屏、两套输入、双方独立 HUD 和升级选择。
- 敌人能够选择最近存活玩家；所有玩家死亡才失败。
- 阵亡玩家在队友清波后于下一波以部分生命复活。
- 升级系统加入普通、稀有和史诗稀有度。
- 初期仅使用临时 Tiny Swords 美术和单一战士框架。

## 2026-07-10：局域网联机骨架

- 使用 ENet 接入双人局域网创建、加入和开始游戏流程，端口为 UDP `24567`。
- 同步双方输入、选角和独立升级选择。
- 使用固定随机种子降低两端独立运行战斗时的差异。
- 新增 Windows 导出与防火墙放行工具。
- 明确当前实现不是完整房主权威；敌人、伤害、死亡和阶段仍需要快照同步与纠偏。
- 代码开始拆分为 `game_rules.gd`、`wave_manager.gd`、`upgrade_manager.gd`、`combat_manager.gd` 和 `player_roster.gd`。
- 建立逻辑、职业战斗和生命周期无画面测试。

## 四职业模块化

- 从单一战士扩展为战士、弓箭手、长枪和法师四个职业。
- 建立统一 `CharacterCombat` 接口，每个职业拥有独立普攻、Q、E、F 模块。
- 升级池拆分为通用升级与技能升级。
- 战士早期技能为冲击波、三方向攻击和两把旋转刀。
- 弓箭手早期技能为直线攻击、三箭散射和普通箭雨。
- 长枪早期技能为突刺、横扫和拒马区域。
- 法师早期仅有远程普攻，Q/E/F 曾暂时保持空白。

## 原创角色美术阶段

- 制定 `ART_STYLE_GUIDE.md`，统一 192×192 单元、角色比例、色板、光源和 y=138 脚底基准。
- 依次完成法师、战士、长枪和弓箭手的原创概念图及九套基础动画。
- 建立高清立绘、像素母版、全套动画、色键清理、锚点对齐和自动验证流程。
- 经历并修复尺寸不一致、锚点漂移、武器被误删、色键残片和动作间体型变化问题。
- 新增 `validate_character_sprites.py`、`clean_character_sprites.py` 和 `normalize_sprite_sheet.py`。
- 公共动画状态统一为 Death、Hit、Cast/Attack、Dash/Defend、Run、Idle 优先级。

## 角色体型和残片修正

- 战士 Idle 回退并稳定为确认版本，修复 Run 与 Idle 的锚点和体型切换。
- 长枪以 Idle 为母版重新统一全套动画尺度，清除攻击帧远距离残片并修复 Run 朝向。
- 弓箭手 Idle 横向加宽，使其接近 Run 的正常身体宽度。
- 四职业增加独立运行时显示缩放，使联机同屏体型更接近。
- 自动检查从简单连通块面积判断，发展为同时检查真实像素距离、合法大型装备、脚底和职业专项身体比例。

## 公共战斗状态与操作修正

- 普攻和技能从按键瞬间结算改为在动作第 3 个可见帧触发。
- 接入公共 Cast、Hit 和 Death；死亡停留最终帧，复活恢复 Idle。
- 防御改为点按不影响移动，持续按住逐步减速并最终停止。
- 新增游戏内“返回主菜单”按钮，方便连续职业测试。
- 修复弓箭生成点、鼠标瞄准方向与真实投射物轨迹不一致的问题。

## 2026-07-13：四职业基础技能定稿

### 战士演进

- 第一版战士 F 使用两把环绕刀。
- 中间方案曾尝试团队壁垒、队友减伤、代伤和跟随范围伤害。
- 该方案因为依赖队友且与长枪持续范围大招相似而废弃。
- 最终战士为攻防一体：Q 聚怪嘲讽并减伤，E 移动防御、强化反伤和近身震击，F 使用三把实体环绕刀。
- 最终 F 只强化战士本人，提供接触伤害、投射物拦截、个人减伤和防御时加速旋转。

### 弓箭手演进

- Q 从瞬发直线攻击改为按住蓄力、松开发射。
- E 从三箭散射改为单体猎杀标记。
- F 改为对同一目标连续命中时逐渐提高伤害的集中箭雨。
- 联机输入增加技能按住/松开和鼠标目标同步。

### 法师演进

- Q/E/F 从占位实现改为最大距离爆炸火球、受限距离持续区域和元素风暴。
- 法师重新进入技能升级池。

### 长枪演进

- Q 定为近距离宽范围横扫。
- E 定为限制最大距离的突进回旋。
- F 定为跟随角色的近距离持续旋转扫击，并保持较长冷却。

## 第一阶段技能特效

- 使用 Godot 原生线条、圆环和半透明区域建立可读性优先的代码特效。
- 特效与伤害系统分离，只读取判定范围和伤害时机。
- 接入弓箭蓄力环、猎杀标记、箭雨轨迹、战士嘲讽连线、反击光线、长枪枪风和法师持续区域脉冲。
- 正式像素技能特效延后到技能行为和数值稳定之后制作。

## 2026-07-15：UI 组件拆分与布局重排

- 将主菜单、选角、战斗 HUD、升级和结算从 `main_game.gd` 中拆为 `scenes/ui/` 场景与 `scripts/ui/` 组件；主游戏脚本保留状态协调、联机和数据绑定职责。
- 统一 Verdant 主题的控件安全高度，修复边框裁切、文字越框和选角卡片位置过高的问题。
- 重排单人和联机选角卡片，接入四职业真实 Q/E/F 图标与更清晰的已选状态标记。
- 为四职业新增横向选角卡面 `*_card_select_redraw_v1.png`；旧版卡面继续保留在独立映射中，未删除或覆盖。
- 高清选角卡面改用线性 mipmap 缩放，像素 UI 仍保持最近邻过滤。
- 战斗 HUD 收紧到 440 宽的紧凑布局，减小生命条、动作槽和间距，降低战斗画面遮挡。
- 新增并扩充 `ui_theme_checks.gd`，覆盖控件边界、文字安全高度、卡面资源、纹理过滤、技能图标和战斗 HUD 尺寸。
- 本轮不由 Codex 启动或查看运行画面；截图问题和未完成的视觉验收记录在 `design-qa.md`，最终视觉确认交由玩家执行。

## 2026-07-15：主游戏安全边界拆分

- 将权威快照构建、应用及联机实体创建从 `main_game.gd` 移至 `scripts/network/authority_sync.gd`，保留原 RPC 入口与兼容包装方法。
- 将静态地图生成、通用战斗表现和输入动作注册分别移至 `map_builder.gd`、`effect_manager.gd` 与 `input_setup.gd`。
- `main_game.gd` 继续作为状态、波次、胜负、RPC 和组件编排入口；未改动高风险的 RPC 房间流程、战斗结算、玩家状态机和敌人 AI 边界。
- 拆分未改变战斗规则或资源，使用无画面解析、生命周期和职业战斗检查验证行为。

## 2026-07-15：配置、动画与流程模块拆分

- 将七类普通敌人与 Boss 的基础数值、成长倍率和特殊参数集中到 `enemy_archetypes.gd`。
- 将玩家和敌人的动画资源路径、帧数、间隔、缩放与色调移至独立动画目录；控制器保留动画状态切换和战斗事件帧。
- 将波次敌人组合、出生点、模式倍率与 Boss 增援创建移至 `wave_spawner.gd`，房主生成判断和波次状态仍由主游戏脚本负责。
- 将升级抽取、槽位状态、选择校验与应用移至 `upgrade_session.gd`，所有 RPC 注解与界面流程保持原位。
- 玩家动作状态机、敌人 AI 主循环、持续技能区域和胜负清理流程未在本轮拆分。

## 当前归档结论

- 四职业基础技能和第一阶段特效已确认。
- 游戏 UI 已完成第一轮组件拆分、选角重排和战斗 HUD 紧凑化；后续只根据新的明确反馈继续调整。
- 刺客继续延期。
- 下一阶段转向四职业技能专属行为升级和第一轮职业平衡。
- 后续当前状态变更应更新 `process.md`；只有被替换或完成的阶段才追加到本历史文件。
